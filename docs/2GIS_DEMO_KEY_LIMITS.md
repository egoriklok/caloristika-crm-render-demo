# 2GIS Demo Key Limits For AI Agents

Источник: лимиты demo key в 2GIS Console, зафиксированные оператором по
скриншотам от 2026-06-13. Эти значения должны считаться жестким ограничением
для всех AI-agent workflows, пока не подключен production key с отдельным
договором/квотой.

Файл-правило для агентов: `docs/2GIS_DEMO_KEY_LIMITS.md`.

## Главное правило

AI-агент не должен использовать demo key 2GIS как инструмент массового сбора
лидов. Demo key нужен для точечной проверки карточки компании, адреса, публичных
контактов, сайта и признаков численности перед КП.

По умолчанию:

- один запуск агента: не больше 10 компаний или 10 кандидатов 2GIS;
- массовые сценарии только через `dry_run` и cache;
- `force_refresh` использовать только для 1-3 выбранных компаний;
- перед большим enrichment/outreach нужен отдельный production key или ручное
  подтверждение лимитов;
- при ошибках 429/403/blocked агент останавливает 2GIS-запросы и пишет задачу
  менеджеру, а не пробует новый demo key.

## Search APIs

Для `Places API`, `Geocoder API`, `Suggest API`, `Categories API`, `Regions API`
и `Markers API`:

| Лимит | Действие |
| --- | --- |
| 600 запросов в минуту | остановка |
| 1000 запросов в месяц | блокировка |

CRM использует прежде всего `Places API` через `DGIS_API_KEY`.

## Agent Guardrails

- Не обходить блокировку созданием новых demo keys.
- Не распараллеливать 2GIS-запросы на demo key.
- Не запускать nightly/bulk enrichment на demo key.
- Не использовать Navigation APIs для массового расчета маршрутов на demo key.
- Для расстояний в CRM сначала использовать сохраненные координаты, адресные
  оценки и cache; live 2GIS-запрос только для выбранных лидов.
- Любой результат 2GIS сохранять с provenance в `company_enrichment_sources`.
- Если monthly usage неизвестен, считать остаток маленьким и ограничиваться
  ручной проверкой 1-3 компаний.
