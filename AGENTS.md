# Lunch-UP CRM Agent Instructions

## gstack for Codex

Use this section when work in this CRM benefits from gstack-style planning,
review, QA, or documentation.

Installed runtime:

- gstack repo: `/home/egori/.gstack/repos/gstack`
- Codex skills generated by gstack: `/home/egori/.codex/skills/gstack-*`
- Browser runtime: `/home/egori/.gstack/repos/gstack/browse/dist/browse`

Run gstack-related checks from WSL/bash. This project lives on a Windows
OneDrive path, but the working AI substrate for command execution is WSL.

Do not run `gstack-team-init` in this CRM folder right now. The project folder
is not a Git checkout, and the upstream team initializer requires a Git repo and
generates Claude-only team-mode files. Use the Codex adaptation below instead.

## Routing

For CRM product, engineering, design, and QA work, use only the relevant gstack
flows:

- Product or scope review: `gstack-plan-ceo-review`
- Architecture or cross-module change: `gstack-plan-eng-review`
- UI or miniapp review: `gstack-plan-design-review`
- Implementation spec: `gstack-spec`
- Bug investigation: `gstack-investigate`
- Code review: `gstack-review`
- Browser QA: `gstack-qa` or `gstack-browse` when the built-in Codex Browser
  plugin is not enough
- Docs and release notes: `gstack-document-generate`, `gstack-document-release`
- Small safety pause before risky actions: `gstack-careful`, `gstack-guard`

Avoid these by default unless the user explicitly requests deployment, release,
or external infrastructure work:

- `gstack-ship`
- `gstack-land-and-deploy`
- `gstack-canary`
- `gstack-setup-deploy`
- `gstack-sync-gbrain`

## Local Validation

Before claiming CRM work is done, prefer this order:

```bash
npm run gstack:check
npm run verify
npm run build
```

For frontend or miniapp changes, also open the relevant local route in the Codex
Browser plugin. The default local app port is `3011`.

## Project Constraints

- Do not expose secrets from `.env.local` or Telegram/2GIS/DaData config.
- Do not mutate deployment, Telegram, or webhook state without an explicit user
  request.
- Keep gstack as methodology and tooling. Do not replace existing project
  scripts when `npm run verify`, smoke tests, or the built-in Browser plugin are
  already the direct fit.
